###### A.1.2.2.2.6 - The Core Type [Type Specification]

[See below]

**Components**:

"Content": The content Component is very flexible and defines in human-readable language core principles, rules or processes, or subcomponents thereof.

**Doc Identifier Rules**:

Core Documents follow the Document Identifier Rules of Primary Documents. Core Documents have Document Identifiers that are 4 layers or deeper in the Document Tree, and cannot contain 0's [zeros]. Within these constraints, Core Documents can have whatever Document Identifier that is useful for their purpose.

**Additional Logic**:



**Type Category**:

Primary Document

**Type Name**:

Core

**Type Overview**:

The Core Type is the basic building block of the Primary Documents. Core Documents flexibly specify the core principles, rules and processes required to fulfil the specifications made by the Immutable Documents with a focus on clarity, practicality and applicability.
